# NOVA — состояние проекта и план вперёд

Обновлено: 2026-07-03 (конец первой большой сессии разработки).

## Что такое NOVA

Персональная голосовая ИИ-компаньонка: видит экран, комментирует сама, ведёт
голосовой диалог, без цензуры, self-hosted. Голос — Мита из MiSide.
Спека: `docs/specs/2026-07-03-nova-companion-design.md`.
Планы: `docs/plans/` (этап 1 — скелет, этап 2 — первый голос; оба ВЫПОЛНЕНЫ).

## Текущее состояние: РАБОТАЕТ

Этапы 1 и 2 завершены. Живой прогон прошёл: NOVA говорит клонированным
голосом, видит экран (описывает рабочий стол, YouTube, обои), помнит диалог
в рамках сессии, комментирует сама. Все ~70 тестов зелёные.

### Архитектура (кратко)

- **Клиент (ноут, Windows)**: `nova/client/` — захват экрана и микрофон в
  ОТДЕЛЬНЫХ процессах (нативные DLL конфликтуют в одном), детектор смены
  сцены, стриминговое воспроизведение, хоткеи (pynput), реконнект.
- **Сервер (аренда GPU на Vast.ai)**: `nova/server/` — FastAPI-оркестратор,
  vLLM с Qwen3-VL-30B-A3B-FP8 (мозг+глаза, порт 5000), faster-whisper
  large-v3-turbo (уши), XTTS-v2 (голос, клон из voice_sample.wav).
- Связь: WebSocket, JSON + base64, токен-аутентификация (NOVA_TOKEN).

### Инфраструктура и операции

- Секреты в `.env` (не в git): VAST_API_KEY, NOVA_TOKEN.
- Управление сервером: `uv run python scripts/vast.py up|status|down [--destroy] [--write-config]`
- Сейчас есть ОСТАНОВЛЕННЫЙ инстанс id=43664615 (RTX 6000 Ada, Австралия,
  $0.516/ч, ssh2.vast.ai:24614) с прогретым кэшем моделей — `up` поднимет
  его за 2–3 минуты. Хранение диска ~$0.15/день. Баланс Vast: ~$13.85.
- SSH-доступ к инстансу: ключ `~/.ssh/nova_vast` (уже прикреплён).
- Деплой обновлений на живой инстанс:
  `ssh -i ~/.ssh/nova_vast -p <port> root@<host> "cd /workspace/NOVA && git pull -q; pkill -f '[n]ova.server.main'; pkill -f '[i]dle_watchdog'; sleep 1; (setsid nohup bash /root/runner.sh > /var/log/runnerN.log 2>&1 < /dev/null &)"`
  (`/root/runner.sh` лежит на инстансе: подхватывает env из /proc/1/environ,
  ждёт vLLM, стартует оркестратор + автостоп-вотчдог).
- Логи на инстансе: /workspace/vllm.log, /workspace/nova.log,
  /var/log/onstart.log, /var/log/runnerN.log.
- Автостоп: 15 минут без клиентов → инстанс останавливает себя сам.

### Грабли, которые уже пойманы (НЕ наступать заново)

1. Vast API — солянка версий: список инстансов GET /api/v1/instances/,
   а состояние/удаление/ssh — на /api/v0/. Уже учтено в scripts/vast.py.
2. Фильтры выбора карты (все в pick_offer): не Китай (нет доступа к
   HF/GitHub), compute_cap >= 800 (FP8 требует Ampere+), cpu_arch amd64
   (наш образ x86), надёжность >= 0.98, канал >= 200 Мбит, диск >= 80 ГБ.
3. На инстансах НЕТ /workspace по умолчанию — onstart.sh делает mkdir.
4. XTTS ломается на русских фразах длиннее ~182 символов — сервер режет
   текст по предложениям (split_for_tts) и синтезирует максимум 5.
5. Повторы в диалоге («целая палитра» N раз) — лечатся presence_penalty=0.8
   + frequency_penalty=0.6 в qwen_llm + правила в системном промпте.
6. Обрезка ответа по токенам рвала фразы — trim_to_sentence отсекает хвост.
7. Звуковая петля (слышит себя из колонок) — клиент глушит микрофон, пока
   NOVA говорит (+1 сек после). В наушниках проблемы нет вовсе.
8. В голосовые ответы НАДО прикладывать свежие кадры (2 шт) — иначе она
   сочиняет про экран. Уже сделано (build_reply_messages).
9. На ноуте: dxcam/PortAudio/onnxruntime в одном процессе = access
   violation → захват и микрофон живут в дочерних процессах. Библиотека
   keyboard битая — заменена на pynput.
10. Windows CRLF ломает bash-скрипты — .gitattributes: *.sh eol=lf, плюс
    vast.py нормализует onstart при отправке.

### Известные проблемы (не критичные)

- Голос XTTS: роботность и лёгкие искажения — главный кандидат на замену
  (см. план «Голос 2.0» ниже).
- Между предложениями длинного ответа бывают паузы синтеза.
- Проактивные комментарии пока редкие/простые — тюнить болтливость в
  profiles/*.yaml по вкусу.

## ПЛАН ВПЕРЁД (по приоритету)

### Следующая сессия: «Голос 2.0» — замена XTTS

Пользователь нашёл готовый клон голоса Миты отличного качества на платформе
fish.audio: модель «Mita Miside (Russian voice)», id в URL:
`6dc11f3f67a543f6ad4537a4a347e224` (страница:
https://fish.audio/ru/m/6dc11f3f67a543f6ad4537a4a347e224/).

Решение (согласовано): **вариант А — self-hosted**.
- Поставить на инстанс открытую модель Fish Audio **OpenAudio S1-mini**
  (репо fish-speech), вместо/рядом с XTTS, за тем же интерфейсом TTSModel
  (новый класс в nova/server/models/, конфиг через env NOVA_TTS=xtts|fish).
- Референс для клона — те же дорожки: `voices/*.ogg` (35 шт, локально,
  не в git) / `personas/nova/voice_sample.wav`.
- Сравнить с XTTS на слух, оставить лучшее.
- Запасной вариант Б: облачный API fish.audio с их готовой моделью Миты
  (максимальное качество, но платно/зависимость/приватность — пользователь
  предпочитает self-hosted).
- Альтернатива/дополнение: дообучение (fine-tune) XTTS или S1-mini на всех
  35 дорожках — субтитры сгенерирует Whisper автоматически (~1–2 ч GPU).

### Этап 3 (по спеке): память и знания

Идеи, согласованные с пользователем 2026-07-03 (обсуждение в сессии):
- **Умное дослушивание (семантический эндпоинтинг)** — сервер после ASR
  смотрит, завершена ли мысль (виспер ставит пунктуацию; «висячие»
  окончания: союзы/предлоги/«эээ»/многоточие → ждать продолжения 3–5 с,
  склеить со следующим сегментом; жёсткий потолок ~8 с). Всё на сервере,
  хрупкий аудио-клиент не трогаем. Мычание «эээ/ммм» VAD уже держит как речь.
  Поверх автоматики — голосовые маркеры пользователя: обращение с вопросом
  («...что думаешь, Нова?») = мысль закончена, отвечать на всё накопленное;
  «Нова, слушай» / «погоди, думаю» = режим накопления без таймеров (молчит
  и буферизует куски, паузы хоть минутные) до явного обращения.
- **Обучение манере/юмору — конвейер дистилляции**: источник (VOD стримера,
  свои сессии, позже Discord) → whisper-расшифровка → LLM-выжимка (словарь
  сленга, паттерны шуток, реакции) → в персону/память сразу, пары
  «ситуация→реплика» в корпус для LoRA (этап 6). Пассивный просмотр сам по
  себе не учит — учит ночной конспект.
- **Совместный просмотр** (аниме/фильмы/игры) — главный источник эпизодической
  памяти («что смотрели, с чего орали»); для стримов/видео нужен захват
  системного звука (WASAPI loopback) — сейчас клиент слышит только микрофон.
- **Discord (этап 4) как источник**: бот отдаёт отдельную дорожку на каждого
  участника — размеченные диалоги бесплатно; при работе в Discord локальный
  микрофон выключать (иначе пользователь слышен дважды). Друзья должны знать.

- **ВАЖНО (решение принято): память живёт на ноуте, не на инстансе.**
  `vast.py down` стягивает `memory/`, `knowledge/`, `data/feedback.jsonl`
  с инстанса (scp по уже настроенному ключу ~/.ssh/nova_vast), `vast.py up`
  заливает обратно. Иначе `--destroy`/смена карты уничтожает всё нажитое.
  (Сейчас на спящем инстансе уже лежит feedback.jsonl первого дня —
  забрать при первом же подключении.)
- Память сессий: конспект диалога при отключении → `memory/`, загрузка при
  старте (она будет помнить прошлые разговоры — пользователь этого явно ждёт).
- Сжатие переполненной истории сессии в конспект («бесконечная» память).
- Поднять max_model_len до 32768 (влезает; при желании 64k через
  --kv-cache-dtype fp8).
- RAG-база `knowledge/` (Chroma) + команда «запомни: ...».
- Web-самообучение: инструмент поиска, «изучи X» → конспект в базу.
- Игровые профили: minecraft, universal_game (+ гайд для новых).
- Голосовая регулировка болтливости («говори почаще/пореже»).

## Шпаргалка запуска (для пользователя)

    uv run python scripts/vast.py up          # поднять сервер (2-3 мин)
    uv run python scripts/vast.py status      # готовность и баланс
    uv run python -m nova.client.main         # клиент (говорить!)
    uv run python scripts/vast.py down        # закончил — погасить

Хоткеи: Ctrl+Alt+C — прокомментируй; Ctrl+Alt+M — mute; Ctrl+Alt+P — пауза;
Ctrl+Alt+↑/↓ — оценка реплики.
