# NOVA — персональный ИИ-компаньон: дизайн

**Дата:** 2026-07-03
**Статус:** согласован с пользователем, ждёт финального ревью

NOVA = Neural Optimization Virtual Assistant.

## 1. Цель

Персональный ИИ-компаньон, который в реальном времени видит экран ноутбука/пк
(аниме/фильмы, майнкрафт, десктоп, позже — экшен-игры), **сам подмечает события**
и комментирует их голосом без запроса, ведёт голосовой диалог с пользователем
и его друзьями, обучаем и без цензуры. Не для стриминга — для себя.

## 2. Требования и ограничения

- **Без цензуры — оба смысла**: дерзкий стиль (мат, троллинг) И свобода по любому
  контенту на экране. Следствие: только self-hosted открытые модели.
- **Общение**: двусторонний голосовой диалог + проактивные комментарии по её
  инициативе. Позже — участие в голосовом канале Discord.
- **Железо пользователя**: ноут Ryzen 7 5800H, встроенная графика (AMD Radeon Vega), FullHD 120 Гц.
  Локально тяжёлые модели не потянет → тонкий клиент + арендованный GPU.
- **Бюджет**: ориентир ~£25/мес, допустимо до £50/мес. Качество в приоритете.
- **Использование**: 3–5 часов в день, почасовая аренда (вкл/выкл сервер).
- **Задержки (цели)**: ответ на фразу — 1–1.5 с; проактивный комментарий — 1.5–2.5 с
  после события.
- **Детали**: должна разбирать мелкие элементы экрана (курсор!, что показывает курсор (область которую обводит пользователь курсором), HUD, мелкий текст)
  без избыточной нагрузки на ноут и GPU.
- **Первый сценарий**: аниме/фильмы + десктоп → затем майнкрафт → затем
  экшен-игры (Fortnite, Marvel Rivals, CS через GeForce Now, когда заработает).
- **Персона**: стартовый характер — дерзкая «Светка»-стайл (женский голос, наглая,
  троллит, матерится), всё редактируемо.

## 3. Архитектура

Два компонента, связь по WebSocket:

```
НОУТ (тонкий клиент, Python)             GPU-СЕРВЕР (аренда, 48 ГБ, Docker)
┌──────────────────────────────┐        ┌───────────────────────────────────┐
│ Захват экрана (dxcam, 30FPS) │─кадры─▶│ Оркестратор (FastAPI + WebSocket) │
│ Детектор событий (opencv)    │        │  ├─ Qwen3-VL 30B-A3B — мозг+глаза │
│ Микрофон + VAD (silero)      │─голос─▶│  ├─ faster-whisper — уши          │
│ Динамики (stream playback)   │◀─аудио─│  ├─ XTTS-v2 — голос NOVA          │
│ Трей + хоткеи + настройки    │        │  ├─ RAG (Chroma) — знания/память  │
│ Discord-бот (py-cord)        │◀──────▶│  └─ Web-search tool — самообучение│
└──────────────────────────────┘        └───────────────────────────────────┘
```

Ноут делает только дешёвую работу (захват, сравнение кадров, аудио I/O);
ожидаемая нагрузка CPU ~10–15%. Все нейронки — на сервере.

## 4. Сервер

### 4.1 Модели

| Роль | Модель | VRAM (прибл.) | Примечания |
|---|---|---|---|
| Мозг + глаза | **Qwen3-VL-30B-A3B-Instruct, FP8** через vLLM | ~30 ГБ + KV | MoE: активны 3B из 30B → умная и быстрая. Одна модель и видит, и говорит, и ведёт диалог |
| Альтернатива | Qwen3-VL-32B dense (конфигом) | больше | Максимум ума, медленнее — сравним на практике |
| Уши | faster-whisper large-v3-turbo | ~1.5 ГБ | Русский — отлично; распознаёт всех участников |
| Голос | XTTS-v2, стриминг | ~3 ГБ | Клонирование голоса из 10-сек сэмпла; начинает звучать до конца генерации |

- **Цензура**: снимается системным промптом персоны. Если базовая модель
  упирается — drop-in замена на abliterated-версию с HuggingFace (та же
  архитектура, снятые отказы).
- Один GPU 48 ГБ вмещает всё одновременно, без выгрузки моделей.

### 4.2 Кадровый пайплайн («видит как человек, не сжигая GPU»)

Двухуровневая система:

1. **Локальный детектор (ноут, 30 FPS, CPU)**: сравнение кадров (opencv/numpy) —
   движение, вспышки, смена сцены, изменения в заданных зонах. Видит всё,
   стоит копейки.
2. **Умная выборка в облако**:
   - спокойная сцена → **1 кадр/сек в полном 1080p** (мелкие детали читаемы);
   - событие → **burst 5–8 последовательных кадров** за ~1 сек, отправляются
     как последовательность — модель видит динамику («кинул флешку → взрыв →
     зашёл на точку»), а не случайный скриншот.
3. **Зоны внимания** (пер-профиль): указанные области (килл-фид, радар, счёт)
   дополнительно отправляются увеличенными вырезками поверх основного кадра.

Устаревшие кадры не буферизуются — если канал/модель заняты, старое
выбрасывается, свежее в приоритете.

### 4.3 Диалог и проактивность

- **Реактивный путь**: голос → whisper → LLM (+RAG, +история) → XTTS → стрим на ноут.
- **Проактивный путь**: событие от детектора → модель получает burst + контекст
  и решает, комментировать ли. Принцип: **фильтр = защита от спама и повторов,
  НЕ цензура инициативы**. По умолчанию смещение в сторону «говорить»; молчит,
  только если событие — дубль уже прокомментированного или на экране ничего.
- **Болтливость** — ручка на профиль (от «редко и метко» до «не затыкается») +
  cooldown между репликами. (кулдаун можно менять, например саму ее попросить говорить чаще или наоборот реже)
- **Перебивание** — переключаемая опция, по умолчанию ВЫКЛЮЧЕНА (она договаривает,
  даже если пользователь заговорил).
- Контекст модели: системный промпт персоны + описания последних кадров +
  история диалога + RAG-выдержки + память сессий.

### 4.4 Знания, память, обучение

Четыре механизма, от мгновенного к медленному:

1. **Память сессий (авто)**: по завершении сессии диалог и события сжимаются в
   конспект → `memory/`. При следующем запуске NOVA помнит прошлые сессии.
2. **RAG-база** `knowledge/` (markdown): лор игр, синопсисы, мемы, заметки.
   Команда «запомни: …» — пишет заметку сама.
3. **Самообучение через интернет**: web-search инструмент на сервере. Видит
   незнакомую игру/аниме или получает команду «изучи X» → ищет, читает,
   сохраняет конспект в `knowledge/`. Автоматическое наполнение RAG.
4. **LoRA-дообучение (батч)**: хоткеи 👍/👎 копят датасет из её реплик с оценками;
   периодически (вручную, раз в месяц-два) запускается тренировка на той же
   48 ГБ карте → стиль и «инстинкты» сдвигаются под вкус пользователя.
   Реального «обучения весов на лету» не существует ни у кого — живой эффект
   дают механизмы 1–3.

### 4.5 Персоны

`personas/nova/`:
- `system_prompt.md` — характер (стартовый пресет: дерзкая «Светка»-стайл),
  редактируется обычным текстовым редактором;
- `voice_sample.wav` — референс для клонирования голоса;
- `settings.yaml` — болтливость по умолчанию, параметры TTS.

Новая персона = копия папки.

## 5. Клиент (ноут, Windows)

- **Python-приложение**: иконка в трее, консольный лог. Тяжёлая работа — в
  нативных библиотеках (dxcam → DXGI duplication, opencv, sounddevice).
  Если детектор упрётся в CPU — перенос на Rust, но не ожидается.
- **Аудио-шина (абстракция источников/выходов)** — ключ к Discord-режиму:
  - источники: локальный микрофон | per-user потоки Discord;
  - выходы: локальные динамики | голосовой канал Discord.
- **VAD** (silero-vad): слышит постоянно, в облако летит только реальная речь.
- **Хоткеи (v1 — в конфиге, позже GUI)**: mute NOVA, «прокомментируй сейчас»,
  пауза наблюдения, 👍/👎, переключение профиля, [позже] «шепнуть NOVA».
- **GUI настроек (этап 5)**: PySide6-окно — назначение биндов кликом, громкость,
  болтливость, выбор профиля/персоны, редактор промпта.

### 5.1 Профили

`profiles/*.yaml` — настройки на тип контента:
- `universal_game` — работает с любой игрой из коробки (модель сама распознаёт игру);
- `anime`, `desktop`, `minecraft` — стартовый набор;
- далее по шаблону: Fortnite, Marvel Rivals и т.д.

Профиль задаёт: частоту кадров/чувствительность детектора, зоны внимания,
дополнение к промпту, болтливость, привязку файлов знаний. Пишется гайд
«профиль под новую игру за 15 минут»; первые игровые профили делаем вместе.

## 6. Discord-режим (этап 4)

- **Бот-аккаунт** через Discord Developer Portal (селфбот на живом аккаунте
  запрещён ToS — не делаем). Библиотека: py-cord (voice receive/send).
- NOVA заходит в голосовой канал: слышит **каждого участника отдельным потоком
  с ником** (различение голосов из коробки, без диаризации) и говорит в канал —
  слышат все.
- **Дедупликация владельца**: в Discord-режиме голос пользователя берётся только
  из его дискорд-потока, локальный микрофон отключается. Позже — хоткей
  «шепнуть NOVA» (локальный микрофон, канал не слышит).
- Экран она по-прежнему видит только с ноута пользователя.

## 7. Инфраструктура и деньги

- **GPU**: Vast.ai (или RunPod как запасной), **RTX A6000 48 ГБ**, ~$0.45/час.
  При 4–5 ч/день ≈ **£40–48/мес**. Всё в Docker-образе.
- **Постоянный volume** (~£4/мес): веса моделей, знания, память — при старте
  инстанса ничего не перекачивается.
- **Скрипты `up` / `down`**: поднять/погасить одной командой; старт ~3–4 мин.
- **Автостоп**: клиент отключён 15 минут → сервер гасит себя сам.
- **Спенд-трекер**: учёт расходов за месяц, предупреждение у порога лимита.

## 8. Обработка сбоев

- Клиент: автопереподключение с backoff; статус сервера виден в трее.
- Кадры при заторе выбрасываются (свежесть важнее полноты).
- Отказ TTS → ответ текстом в лог/окно.
- Сервер: docker restart policy; watchdog при OOM/зависании модели.
- Автостоп защищает от «забыл выключить».

## 9. Тестирование

- **MOCK_MODELS=1**: сервер отдаёт заготовленные описания/аудио — весь пайплайн
  гоняется локально, без GPU и денег.
- **Фикстуры**: записанные клипы (аниме-сцена, майнкрафт, десктоп) → офлайн-прогон
  «что бы она сказала» для тюнинга триггеров и промптов без аренды.
- **Метрики задержек** по стадиям (захват → отправка → модель → первый звук)
  пишутся в лог каждой сессии.

## 10. Этапы

1. **Скелет** — клиент + сервер с заглушками, локально, бесплатно.
2. **Первый голос** — аренда GPU, реальные модели: совместный просмотр аниме,
   голосовой диалог, проактивные комментарии. ← первый «вау»-момент
3. **Профили и знания** — майнкрафт, universal_game, RAG, web-самообучение,
   тюнинг болтливости.
4. **Discord** — бот в голосовом канале, друзья слышат, различение участников.
5. **GUI настроек** — окно биндов, редактор персоны.
6. **Дообучение** — LoRA по оценкам 👍/👎, полировка клона голоса.

## 11. Что осознанно НЕ делаем (YAGNI)

- Стриминг-интеграции (OBS, чат Twitch) — не для стрима.
- Аватар/визуал персоны — только голос (можно вернуться позже).
- Управление игрой/компом с её стороны — она смотрит и говорит, не действует.
- Диаризация голосов вне Discord — в Discord-режиме не нужна, локально
  многоголосье не поддерживаем.
- Реалтайм-дообучение весов — не существует; закрыто механизмами памяти/RAG.
