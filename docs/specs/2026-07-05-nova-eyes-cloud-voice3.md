# Спека: «Глаза в облако + Голос 3.0» (мини-этап перед памятью)

Дата: 2026-07-05. Статус: черновик на согласовании.

## Зачем

1. **Голос.** Бесплатный тариф fish.audio (s2.1-pro-free) умирает в конце
   июля 2026 — облачная Мита D доживает последние недели. VoxCPM2 прошёл
   уши («всё супер шикарно, идеально»): локальный, 48кГц, бесплатный
   навсегда, рецепт готов (scripts/voxcpm2_bench.py). Но ему нужно ~8ГБ
   VRAM, которых на карте нет.
2. **Память (VRAM).** Мозг-VLM резервирует 85% карты, потому что кадры
   экрана стоят ~4К токенов каждый (6 кадров ≈ 24К из 32К контекста).
   Если зрение уедет в Gemini, мозгу останется чистый текст → резерв
   можно опустить до ~0.70 → высвобождается ~7-8ГБ ровно под голос.
3. **Качество глаз.** Gemini смотрит поток кадров «почти как видео» и
   шлёт мозгу короткие сводки — NOVA будет видеть больше, а контекста
   тратить в ~20 раз меньше. Задел под этап 3 (со-просмотр, конспекты).

## Что НЕ меняется

- Мозг: тот же Qwen3.6-27B-FP8, те же веса, тот же ум. Меняется только
  размер зарезервированного KV («забор»), не качество.
- Клиент: захват экрана, детектор событий, протокол — без изменений
  (кадры как шли на сервер, так и идут).
- Персона, эмоции-ремарки, СТТ-страж — без изменений.
- Локальное зрение остаётся аварийным режимом (NOVA_EYES=local), как
  локальный fish у голоса.

## Архитектура

### Глаза: GeminiVision (nova/server/models/gemini_vision.py)

Адаптер, реализующий тот же интерфейс VisionLLM, оборачивает мозг:

- `reply_to_user(text, frames, history)`: если frames непусты — сначала
  Gemini описывает их одним запросом (короткая выжимка «что на экране»,
  1-3 предложения на кадр, свежий кадр подробнее), затем мозг вызывается
  БЕЗ картинок: описания подставляются системной вставкой «[экран: ...]».
- `comment_on_event(event, frames, history)`: аналогично — описания
  вместо jpeg.
- Кэш описаний по хэшу кадра: один и тот же кадр не описывается дважды
  (кадры в deque живут долго, событие может прийти пачкой).
- Промпт Gemini: описывать фактами, без домыслов, замечать изменения
  относительно предыдущего описания (передаём прошлую сводку), на
  русском. Отказ/пустой ответ Gemini → честная вставка «[экран виден
  плохо]» + лог; NOVA не выдумывает.
- Модель: env NOVA_GEMINI_MODEL (default — актуальный Flash-Lite;
  уточнить по доступности на ключе). Ключ: GEMINI_KEY в .env, на боксе
  /workspace/gemini_key (как fish_key).
- Переключатель: NOVA_EYES=gemini|local (default gemini при наличии
  ключа). local = сегодняшнее поведение (кадры в Qwen), обязателен как
  фолбэк на случай цензуры/лимитов/сбоя облака.

### Голос: VoxTTS (nova/server/models/vox_tts.py)

Движок TTSModel по рецепту-чемпиону:

- VoxCPM2 python API в процессе сервера (модель ~8ГБ, load_denoiser=False).
- Перед синтезом каждого предложения:
  1) RUAccent расставляет ударения (нотация U+0301 «про́мах» — проверена;
     env NOVA_VOX_STRESS=0 отключает начисто — страховка stress0);
  2) тег темпа «(Speaking very slowly, at a calm and relaxed pace)»
     приклеивается к тексту (env NOVA_VOX_TAG, пусто = без тега);
  3) генерация с torch.manual_seed(NOVA_VOX_SEED=42).
- Срез наговоренной шапки: whisper СТТ-стража уже транскрибирует каждое
  предложение (guard) — переиспользуем ту же модель с word_timestamps,
  режем до первого слова реплики с запасом 120мс + фейд-ин 20мс.
  СТТ-страж дальше работает как раньше (сверка текста, ретрай).
- Выход 48кГц → как у остальных движков: пик-нормализация 23000,
  ресемпл в формат клиента при необходимости.
- Эмоции-ремарки [sarcastic] и т.п.: VoxCPM2 их не знает — на первом
  этапе strip_markers перед синтезом (мозг продолжает их писать, дисплей
  их не видит и так). Эксперименты со стилем в теге — потом.
- NOVA_TTS=voxcpm. fishcloud остаётся в коде нетронутым (переключение
  назад одной переменной, пока free-тариф жив).

### VRAM-бюджет (карта 48ГБ)

| Жилец | ГБ |
|---|---|
| Qwen3.6-27B-FP8 веса | ~28 |
| KV-кэш при util 0.70 (текст-only) | ~5 |
| whisper large-v3-turbo | ~2 |
| VoxCPM2 | ~8 |
| Запас | ~5 |

runner.sh: --gpu-memory-utilization 0.85 → 0.70, контекст 32К оставить
(текстовому диалогу KV хватает), NOVA_IMG_LIMIT становится неактуален
при NOVA_EYES=gemini (кадры в мозг не ходят).

Порядок старта на боксе: vLLM (0.70) → whisper → VoxCPM2 → готовность.
Если OOM на стыке — первый ход: 0.70 → 0.68 и NOVA_MAX_TOKENS вниз;
второй ход: карта 80ГБ (дороже, крайний случай).

### Деплой

- onstart.sh: venv /workspace/vox (--system-site-packages) + pip install
  voxcpm ruaccent (+ librosa) с ретраями, как fish-venv; прогрев модели.
  ВНИМАНИЕ: воспроизвести грабли RUAccent (monkeypatch token_type_ids
  для onnx — записан в STATUS от F5-эпопеи).
- runner.sh: util 0.70, NOVA_EYES/GEMINI_KEY/NOVA_VOX_* в grep-список
  env, /workspace/gemini_key файл.
- vast.py create_instance: пробросить NOVA_EYES, GEMINI_KEY, NOVA_TTS=voxcpm.

## Риски и фолбэки

1. **Цензура Gemini** на «жёстких» кадрах → вставка «[экран виден плохо]»,
   опционально авто-фолбэк кадра в локальный Qwen (NOVA_EYES_FALLBACK=1).
2. **Лимиты free-тира** (запросов/мин и /день) → кэш описаний, батч
   кадров одним запросом, деградация в local при 429. Замерить реальный
   расход в первый живой день.
3. **Латентность глаз**: +0.5-2с на описание кадра перед ответом мозга.
   Смягчение: описывать кадры заранее (при получении, не при вопросе) —
   если замер покажет, что мешает.
4. **KV при 0.70**: если захрипит на длинных диалогах — 0.72-0.74 всё
   ещё оставляют место голосу (весы: KV против запаса).
5. **Латентность VoxCPM2**: RTF ~0.3 на 4090; на 6000Ada с торч-компайлом
   первое предложение должно успевать раньше облачной очереди free
   (3-6с). Замерить в живом диалоге — критерий ниже.

## Критерии приёмки

1. Живой диалог: голос — VoxCPM2 48кГц, темп «slower», без наговоренной
   шапки, без слоу-мо артефактов; заскоков не больше, чем у облака D.
2. Пауза до первого звука ≤ облака D (сегодня 3-6с из-за очереди).
3. NOVA комментирует экран через Gemini-описания не хуже, чем раньше
   (проверка на реальных сценариях: игра, код, ролик).
4. vLLM стабилен на 0.70 в длинном диалоге (без OOM/захлёба KV).
5. Аварийные режимы: NOVA_EYES=local и NOVA_TTS=fishcloud возвращают
   сегодняшнее поведение одной переменной каждый.
6. Тесты: моки Gemini/VoxCPM2, юниты на кэш описаний, срез шапки,
   ударения, фолбэки; `uv run pytest -q` зелёный.
