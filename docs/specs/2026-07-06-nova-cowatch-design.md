# Спека 3В: Со-просмотр (глаза-видео)

Дата: 2026-07-06. Статус: одобрен вариант А (автомат по движухе).
Опирается на 3А (память в проде: [видела]-канал, дневник, конспект).

## Зачем

Сейчас глаза видят снимками (пульс ~10с + события) — экшн проваливается
между кадрами. Цель: движуху она видит КАК ВИДЕО (клипы с таймштампами
и звуком), статику — по-прежнему дешёвыми кадрами. «Смотрим фильм
вместе» — со звуком: она слышит диалоги героев, а не угадывает по
стоп-кадрам.

## Режимы (вариант А: автомат по движухе)

- **STILL** (статика): как сегодня — пульс-кадры, события, копейки.
- **MOTION** (движуха): клиент пишет экран клипами по NOVA_CLIP_S=15с
  и шлёт серверу; глаза разбирают клип с таймштампами.
- Переключение: детектор клиента считает срабатывания за окно —
  ≥NOVA_MOTION_ON (3 события за 30с) → MOTION; тишина
  NOVA_MOTION_OFF (60с без событий) → STILL.
- **Кино-режим** (принудительный MOTION): хоткей ctrl+alt+v — для
  фильмов, где картинка «спокойная», но смотреть надо непрерывно.
  (Голосовое «Нова, смотрим фильм» — позже, вместе с интентами 3Б.)

## Клиент

- **Захват видео**: кадры уже идут из dxcam — копятся в буфер клипа;
  раз в NOVA_CLIP_S секунд буфер кодируется в mp4 (ffmpeg subprocess:
  rawvideo -> h264, даунскейл до 720p, NOVA_CLIP_FPS=8, faststart).
  ffmpeg обязателен на машине клиента (у Джея есть).
- **Звук системы** (NOVA_CLIP_AUDIO=1): WASAPI loopback через
  pyaudiowpatch (новая клиентская зависимость) — то, что играет из
  колонок, муксится в клип. Не завёлся loopback — клип без звука,
  печать предупреждения, работаем дальше.
- **Отправка**: новый тип протокола Clip(ts, mp4_b64, dur_s). Очередь
  «только свежий» (как LatestSlot у кадров): сервер не успел — старый
  клип дропается, бэклог не копится, пинг в катках не страдает.
  ws max_size 32МБ уже покрывает (клип ~2-5МБ).

## Сервер

- **GeminiEyes.describe_clip(mp4: bytes, hint: str = "") -> str**:
  video/mp4 через inline_data тем же generateContent; промпт: «разбери
  клип с таймштампами: кто что сделал, что сказали (звук!), счёт/итоги;
  по-русски, фактами, 3-6 строк». Модель та же flash-lite.
- **Оркестратор**: Clip -> describe_clip -> строка в дневник через
  on_seen ([видела]-канал 3А, дедуп не мешает — клипы только при
  движухе) + сводка хранится как last_clip для контекста.
- **Комменты**: клип-сводка отдаётся проактивному циклу как событие
  «clip» с текстом сводки — мозг сам решает, комментить ли (PASS
  как всегда; частоту держит существующий движок; судья 3Б потом
  сядет сюда же).
- **Вопросы про экран**: как сейчас (свежий кадр + прицельный промпт),
  ПЛЮС последняя клип-сводка добавляется в контекст вопроса — «что
  сейчас произошло?» отвечается по движухе, а не по стоп-кадру.

## Деньги и лимиты

Видео ~300 ток/с при дефолтном сэмплировании: час движухи ≈ 1.1M ток
≈ $0.11 (flash-lite). Реалистичный месяц (1-2ч движухи/день) — $3-6.
Лимиты Tier 1 (4K RPM / 150K RPD) — наши 4 клипа/мин не видны.

## Крутилки

NOVA_COWATCH=1|0 (рубильник всего 3В), NOVA_CLIP_S=15, NOVA_CLIP_FPS=8,
NOVA_CLIP_AUDIO=1, NOVA_MOTION_ON=3, NOVA_MOTION_OFF=60,
хоткей cinema: ctrl+alt+v.

## Критерии приёмки

1. Замес в Rivals: коммент содержит ход событий («зашёл с фланга,
   съел троих»), а не только итог.
2. Фильм в кино-режиме: пересказ сцены с репликами героев (звук!).
3. Статика (дока/код): расход и поведение как до 3В (STILL).
4. Онлайн-катка: пинг не деградирует (клипы дропаются, не копятся).
5. Дневник: клип-сводки лежат в [видела] с временем; «помнишь замес?»
   назавтра — вспоминает детали.
6. Откат: NOVA_COWATCH=0 возвращает поведение 3А один-в-один.
7. pytest зелёный; юниты: протокол Clip, мото-переключатель клиента,
   describe_clip (мок), оркестратор-клип (мок), дроп-очередь.

## Открытые вопросы

- pyaudiowpatch на его машине (проверим при деплое; фолбэк без звука).
- Качество таймштампов flash-lite на быстром экшне — если жидко,
  ручка NOVA_GEMINI_VIDEO_MODEL=gemini-flash-latest (дороже, зорче).
